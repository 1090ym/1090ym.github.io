# 项目介绍

该博客系统大概包含了以下功能：用户的登录与注册、系统的权限管理、发表编辑博客、博客查阅、博客删除、博客评论、博客点赞、个人信息页展示、 个人信息的修改、首页信息显示如热门标签、热门用户、热门博客、新发表博客以及所有博客列表，同时会做阅读量、评论量以及点赞量的统计， 还支持全文搜索，可以快速查找出想要的内容。当然还为管理员提供了后台管理系统，用来管理与维护博客系统，如用户管理，博客管理等。

博客系统实现的功能有首页的博客列表展示，分类查看博客，博客评论和数据统计，个人信息展示，可以基于博客内容进行搜索，后台管理部分包括管理员用户登录，发表、编辑、删除博客，博客的编辑使用了一个markdown插件。

后端部分，整个项目是采用SpringBoot做为基础框架

  1. 底层采用elastic search和mysql进行存储，使用mysql进行降级
  2. 访问量采用分布式异步方式进行统计更新，
  3. 后端登录功能采用 EasyCaptcha 生成验证码，Redis 优化验证码的保存。
  4. 系统日志采用自定义注解+AOP 方式实现，并使用 ThreadLocal 解决在多线程并发环境下的事务冲突问题。
  5. 数据库字段优化。
  6. 博客使用 Redis 作为缓存，用于分页，数据缓存，布隆过滤器防穿等。

  • 异常处理：SpringBoot框架提供了处理错误页面的方法，在这里，咱们对404、500、error异常页面进行处理，在前端页面templates目录下有error文件夹，SpringBoot可以通过文件夹的名称和错误文件命名的方式找到异常页面

    # 全局异常处理

    • 使用错误页面拦截器，通过定义这个类来拦截所有的异常，它会拦截所有的Controller，将异常信息记录到日志中，然后返回到自定义的错误页面

  • 通过@ControllerAdvance注解，我们可以实现全局的异常控制

# 日志处理

  • 通过一个切面拦截所有的controller类

    • 在方法调用之前记录请求的url，访问者的ip，调用的方法，传递的参数，返回日志对象

  • 返回之后进行拦截，记录返回的值

# 登录拦截器

  • 不能让游客访问到后台管理页面，在这里就需要加一个登录拦截器，指定拦截内容，将访问路径给过滤掉

    • 实现WebMvcConfigurer接口，使用addInterceptors 方法用来注册添加拦截器

  • 通过继承HandlerInterceptorAdapter适配器，重写预处理方法进行拦截

    • 方法：判断session里面是否有用户，没有的话重定向到登录页面，给拦截掉

  • 然后在 WebConfigurer 中的 addInterceptors 中添加拦截器

    • addPathPatterns 用来设置拦截路径，excludePathPatterns 用来设置白名单
