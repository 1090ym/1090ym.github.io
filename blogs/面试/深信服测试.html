# 什么是网关

负责不同的网络之间的通信，在TCP/IP协议也会根据子网掩码(255.255.255.0)判定两个网络中的主机处在不同的网络里。而要实现这两个网络之间的通信，则必须通过网关。如果网络A中的主机发现数据包的目的主机不在本地网络中，就把数据包转发给它自己的网关，再由网关转发给网络B的网关，网络B的网关再转发给网络B的某个主机。网络B向网络A转发数据包的过程。

# Syn_Flood攻击原理

前网络上最为常见的DDoS攻击，也是最为经典的拒绝服务攻击，它利用了TCP协议实现上的一个缺陷，通过向网络服务所在端口发送大量的伪造源地址的攻击报文，就可能造成目标服务器中的半开连接队列被占满，从而阻止其他合法用户进行访问。

它可以方便地伪造源地址，追查起来非常困难。它的数据包特征通常是，源发送了大量的SYN包，并且缺少三次握手的最后一步握手ACK回复。

如果攻击者大量发送这种伪造源地址的SYN请求，服务器端将会消耗非常多的资源来处理这种半连接，保存遍历会消耗非常多的CPU时间和内存，何况还要不断对这个列表中的IP进行SYN+ACK的重试

也就是说在没有任何防护的情况下，每秒发送200个伪造syn包，就足够撑爆半连接队列，从而使真正的连接无法建立，无法响应正常请求。 最后的结果是服务器无暇理睬正常的连接请求—拒绝服务。

解决方法：

  • 提高TCP端口的连接容量
  • 减少半开状态下等待ACK消息的时间或者重试发送SYN-ACK消息的次数
  • 使用一个DDos防御系统，在进行连接时过滤掉那些伪造的IP地址。

# 同步和异步

**同步：**当发起一个同步过程调用时，调用者必须等待调用返回后才能继续往下执行

**异步：**当发起一个异步过程调用时，调用者不必等待调用返回，可以继续往下执行，当调用完成后可以发送通知或者回调给调用者

1、同步的执行效率会比较低，耗费时间，但有利于我们对流程进行控制，避免很多不可掌控的意外情况（单线程） 2、异步的执行效率高，节省时间，但是会占用更多的资源，也不利于我们对进程进行控制（多线程）

# HTTP

请求报文格式：

  1. 请求行
  2. 请求头部
    1. User-Agent
    2. accept
    3. host
    4. cookie
    5. keepAlive
    6. content-type
  3. 请求空行：通知服务器以下不再有请求头
  4. 请求数据

响应报文：

  1. 状态行
  2. 响应头部
  3. 响应空行
  4. 响应数据

GET请求：请求参数和对应的值附加在URL后面

POST请求：POST方法将请求参数封装在HTTP请求数据中，以名称/值的形式出现，可以传输大量数据

HEAD请求：HEAD就像GET，只不过服务端接受到HEAD请求后只返回响应头，而不会发送响应内容。当我们只需要查看某个页面的状态的时候，使用HEAD是非常高效的

PUT请求：上传文件

PATCH请求：对资源进行修改

DELETE请求：删除文件

OPTIONS请求：查询支持的方法

TRACE请求：追踪路径

# HTTP1.0

  1. 使用短连接：浏览器的每次请求都需要与服务器建立一个TCP连接，服务器处理完成后立即断开TCP连接
  2. 服务器不跟踪每个客户端也不记录过去的请求（无状态）

# HTTP1.1

  1. 默认使用长连接：多个请求复用一个TCP连接，避免了连接建立和释放的开销，但服务器必须按照客户端请求的先后顺序依次回送相应的结果，以保证客户端能够区分出每次请求的响应内容。
  2. 支持发送head信息

# HTTP2.0

  1. 使用二进制分帧技术
  2. 使用了多路复用技术：一个连接可以并发处理多个请求
  3. 服务器推送：服务端可以主动把JS和CSS文件推送给客户端，而不需要客户端解析HTML时再发送这些请求
  4. 头部压缩

# NAT

网络地址转换：解决IPv4地址耗尽的问题，将私有IP转换为公有IP，在内部网络中使用私有IP，当内部节点要与外部网络进行通讯时，就在网关（可以理解为出口，打个比方就像院子的门一样）处，将内部地址替换成公用地址，从而在外部公网（internet）上正常使用。

静态NAT：内部的IP地址与公有IP地址是一种一一对应的映射关系

动态NAT：内部有一个公有IP地址池，私有 IP 地址可随机转换为任何指定的公有 IP 地址

**网络地址端口转换 NAPT：**采用端口多路复用方式，内部网络的所有主机均可共享一个合法外部 IP 地址，但是端口号不同

# 快排

    public static void quickSort(int[] nums, int left, int right) {
        if(left >= right) return;
        int i = left, j = right;
        int p = nums[left];
        while (i < j) {
            while (nums[j] > p && i < j) j--;
            while (nums[i] <= p && i < j) i++;
            if(i < j) {
                int tmp = nums[i];
                nums[i] = nums[j];
                nums[j] = tmp;
            }
        }
        nums[left] = nums[i];
        nums[i] = p;
        quickSort(nums, left, i - 1);
        quickSort(nums, i + 1, right);
    }

# 服务端

import java.io.InputStream;
import java.net.ServerSocket;
import java.net.Socket;

public class SocketServer {
    public static void main(String[] args) throws Exception {
        int port = 12345;
        ServerSocket server = new ServerSocket(port);
        System.out.println("等待客户端的连接");
        Socket socket = server.accept();

        InputStream inputStream = socket.getInputStream();
        byte[] bytes = new byte[1024];
        int len;
        StringBuilder sb = new StringBuilder();
        while ((len = inputStream.read(bytes)) != -1) {
            sb.append(new String(bytes, 0, len, "UTF-8"));
        }
        System.out.println(sb);
        inputStream.close();
        socket.close();
        server.close();
    }
}

import java.io.OutputStream;
import java.net.Socket;

public class SocketClient {
    public static void main(String[] args) throws Exception {
        String ip = "127.0.0.1";
        int port = 12345;
        Socket socket = new Socket(ip, port);
        OutputStream outputStream = socket.getOutputStream();
        String s = "hello world";
        outputStream.write(s.getBytes("UTF-8"));
        outputStream.close();
        socket.close();
    }
}
