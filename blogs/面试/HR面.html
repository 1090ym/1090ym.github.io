HR面

mario 2020-10-04 HR面面试

# 缺点

● 当要做一个新任务时可能会有一点急躁，想着把任务完成并达到一个自己想要的结果，我自己的解决办法是尽量每次都把任务细分，制定计划逐步去完成。 ● 作为一个应届生，缺乏一些实战的经验，但是通过一些新人培训来提升自己的实战经验。

# 优势

  • 当有了一个明确的目标后，就会严格按照自己的目标和节奏进行下去。
  • 喜欢看一些书籍，了解底层的实现。

# 前面面试怎么样

面试官人很好，面试体验也挺好的，他问了一些计算机基础相关的问题，这部分我答得还可以，还问了一些java相关的问题，我基本也都答出来了，但是也有一些回答得不是很好的问题，然后面试官也会引导我，让我回忆起相关的知识。

# 有没有其他Offer

暂时没有，但是有一些公司面了HR面，还在等结果

# 自己的未来发展方向

自己对技术的热情到底是怎样的。比如是更多来“解决了问题，探索了未知领域”

我个人的兴趣是后端方向，在未来五年，我打算在后端领域持续积累自己的技术，打磨自己的技术广度和深度，成为一个后端的高级开发，再慢慢成为这个领域的资深专家或者架构师。具体规划是，在公司里认真做好自己的工作，实现自己的价值，一方面在工作中不断积累技术经验，学习一些框架的源码，了解底层知识，看一些前沿的后端相关的paper。

如果喜欢做技术，想一直做技术，他可以这么回答：“未来五年，我打算在后端领域持续深耕，打磨自己的技术广度和深度，能够成为资深的专家或者架构师。

如果将来想转管理的话，可以这么回答：“接下来的两三年内，我希望继续在技术方面多做积累，不断提升解决复杂问题的能力，再往后的话，如果有机会，我想试试技术管理角色

# 加班频率

能接受加班。

# 你是如何学习的

先去了解这个东西是用来干什么的，可以解决什么样的问题，使用于什么样的场景，它的一些优缺点，然后就可以去看一下它的官方文档，然后就可以去一些教学网站，比如慕课网、中国大学慕课、B站，做一些简单的练手项目，基础比较扎实了，就去github上找一些相关的点赞数比较高的开源项目，去看看这些开源项目的源码。最重要的是在学习的过程中要去做学习笔记和总结，一定要写代码。

# 为什么选择这个公司

公司是2016年从豌豆荚分离出来，是一个比较年轻的创业公司，技术氛围会比较活跃，能够参与到一些核心模块的开发，发展空间会比较大，公司SnapTube, Lark Player 等多款广受海外新兴市场用户欢迎的产品，有多家一线投资方，正好也有后端开发的岗位。我个人比较感兴趣

# 你对新工作的期望

希望有一个活跃的技术氛围，能够和团队成员进行良好的沟通和协作，相互请教

良好的发展空间，能够通过自己的努力，提升自己的技术能力和生活水平

能够为公司做出贡献，实现自身价值，完成上级主管对自己的工作预期

能够很好的平衡自己的工作和生活

# 最有成就感的一件事

在做软件工程结课项目的时候，当时要求组队进行，团队里一共有3个人，我作为组长，带领大家开发了一个简单的笔记app（笔记，代办，日程，倒数日），确立项目的概要设计，首先大家每个人的技术基础都不太一样，所以我首先需要确定学习使用哪一个技术来实现我们的目标，然后就去看一下相关的官方文档，我作为小组长要负责项目的总体架构，模块的划分，根据团队成员的能力来分配相关的任务，通过共享文档提交自己的开发进度，当他们完成了自己的任务后，我还需要负责整合，查看是否有一些地方做得不是很好，然后再进行测试，还要负责版本的管理。

我在团队中扮演了一个串联队友的作用，当开发出现停滞时，能够明确团队的目标和方向，当遇到问题时，也会一起解决。

最后当项目完成并通过测试，然后在课堂上进行了答辩演示，得到了老师的良好评价。

印象最深的就是当时大家都对这个项目感兴趣，也很有热情，连续熬了几天晚上，特别是当你投入到自己所热爱的事情中，其实也是一件很幸福的事。

成就感的点：就是在我的带动下，大家都能完成自己负责的那部分任务，讨论问题也很积极，答辩时表现也很好。

技术：要实现的东西不算是非常难，但是有时候遇到一个小Bug，因为不是太了解相关的底层原理，就可能会耗费一定的时间。

非技术点：带动大家按时完成任务，团队成员之间的协作情况。

# 如果有一个有挑战性的事情？你会怎么去做？

确定一下事情的难度，如果是非常难，超出自己的能力之外，根本就无从下手，那我可能需要去寻求一些帮助，知道应该要如何入手，有一个具体的方向。如果不是非常难，用什么工具或技术去解决它，先给自己确立一个具体的思路，每个阶段要完成哪些任务，循序渐进的完成他。
